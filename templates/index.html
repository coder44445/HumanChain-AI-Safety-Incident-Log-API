Incident Log API Documentation

A RESTful API for managing incident logs

GET
/incidents
Retrieve all incidents from the database
curl -X GET https://humanchain-ai-safety-incident-log-api-rak8.onrender.com/
POST
/incidents
Create a new incident in the database
curl -X POST https://humanchain-ai-safety-incident-log-api-rak8.onrender.com/incidents \
-H "Content-Type: application/json" \
-d '{"title": "Server Down", "description": "Main server is not responding", "severity": "High"}'
GET
/incidents/<id>
Retrieve a specific incident by its ID
curl -X GET https://humanchain-ai-safety-incident-log-api-rak8.onrender.com/incidents/1
DELETE
/incidents/<id>
Delete a specific incident by its ID
curl -X DELETE https://humanchain-ai-safety-incident-log-api-rak8.onrender.com/incidents/1

Important Notes

  • Severity levels must be one of: Low, Medium, High
  • All requests and responses are in JSON format
  • Title and description must be strings
  • All endpoints require proper authentication
